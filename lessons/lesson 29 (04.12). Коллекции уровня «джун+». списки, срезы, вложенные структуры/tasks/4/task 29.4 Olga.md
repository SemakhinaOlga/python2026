# Задание 29.4. Коллекции уровня «джун+». списки, срезы, вложенные структуры

## Блок урока
Вложенные структуры. Отрабатываем навигацию по списку словарей и аккуратную выборку вложенных значений.

## Условие задачи
Ты управляешь расписанием мини театральных площадок:
1. Создай список словарей `stages` с ключами `name`, `capacity`, `shows` (список словарей с полями `title`, `start`).
2. Распечатай для каждой площадки строку `Сцена: name. места: capacity`. затем выведи вложенный список спектаклей.
3. Построй список спектаклей, которые начинаются после 18:00, собрав их из всех площадок.
4. Сформируй словарь `title -> name сцены`, чтобы быстро понять где идет постановка.
5. Подсчитай общее количество показов и выведи итоговое число.

## Пример входных данных
```python
stages = [
    {
        "name": "Мансарда",
        "capacity": 45,
        "shows": [
            {"title": "Сны", "start": "17:00"},
            {"title": "Тихий город", "start": "19:30"}
        ]
    },
    {
        "name": "Подвальчик",
        "capacity": 30,
        "shows": [
            {"title": "Лабиринт", "start": "18:15"}
        ]
    }
]
```

## Пример выходных данных
```
Сцена: Мансарда. места: 45
  - Сны в 17:00
  - Тихий город в 19:30
Сцена: Подвальчик. места: 30
  - Лабиринт в 18:15
Вечерние спектакли. ['Тихий город', 'Лабиринт']
Карта спектаклей. {'Сны': 'Мансарда', 'Тихий город': 'Мансарда', 'Лабиринт': 'Подвальчик'}
Всего показов. 3
```

## Критерии успешного выполнения
- Список площадок содержит вложенные списки спектаклей
- Вывод по сценам формируется циклом без ручного обращения к индексам
- Фильтрация по времени использует сравнение строк времени
- Карта `title -> name` собирается программно

## Подсказки
- Итерируй спектакли через вложенный цикл `for show in stage["shows"]`
- Для времени достаточно сравнить строки формата `HH:MM`
- Счетчик показов можно увеличивать на длину списка `shows`

## Частые ошибки и ограничения
- Не забывай проверять, что у сцены есть хотя бы один спектакль
- Не переписывай исходный список при построении карт и фильтров
- Избегай дублирования кода для каждой сцены
- Не используй `print` внутри функций фильтрации. сначала собери данные, потом выводи
