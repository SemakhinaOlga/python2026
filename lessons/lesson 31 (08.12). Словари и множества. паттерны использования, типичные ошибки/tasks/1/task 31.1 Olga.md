# Задание 31.1. Словари и множества. паттерны использования, типичные ошибки
## Блок урока
Словарь как структура быстрого доступа — Блок показывает, как превращать повторяющиеся данные в словарь и мгновенно находить значения по ключу.

## Условие задачи
Сеть кофеен фиксирует, кто последний калибровал кофемолку в каждой точке. Техник быстро решает, куда ехать утром. Создай словарь calibration, где ключ — название кофейни, а значение — дату калибровки. Затем выведи информацию для переменных urgent_location и new_location.

## Пример входных данных
```python
shops = [
    ('Nevsky', '2025-12-04 06:30'),
    ('Kazan', '2025-12-05 08:15'),
    ('Fontanka', '2025-12-03 18:00'),
]
urgent_location = 'Kazan'
new_location = 'Admiral'
```

## Пример выходных данных
```text
Kazan -> 2025-12-05 08:15
Admiral -> не калибрована
```

## Критерии успешного выполнения
- Список пар преобразован в словарь calibration с ключом 'название кофейни'.
- Для обращения к словарю используется метод get с значением по умолчанию.
- Выведены ответы для urgent_location и new_location в заданном формате.
- Код не падает при запросе отсутствующего ключа.

## Подсказки
- Пройдись по списку shops в цикле и наполняй словарь calibration.
- Используй множественное присваивание: key, value = запись из shops.
- Для отсутствующих данных возвращай строку 'не калибрована'.

## Частые ошибки и ограничения
- Не обращайся к словарю через квадратные скобки, если нет проверки на существование ключа.
- Не оставляй ключи кортежами целиком — нужен именно идентификатор.
- Не дублируй вывод: достаточно одной строки на каждый запрос.
