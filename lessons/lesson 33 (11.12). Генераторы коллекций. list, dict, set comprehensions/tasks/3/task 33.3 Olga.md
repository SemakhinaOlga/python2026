# Задание 33.3. Генераторы коллекций. list, dict, set comprehensions

## Блок урока
«Dict comprehensions» учит собирать словари с вычисляемыми значениями в одну строку, сохраняя читаемость.

## Условие задачи
**Сложность**: <span style="color: #f9a825">● средний</span>. примерное время: 15–20 минут

Есть список `projects`, где каждый элемент — словарь с ключами `"title"`, `"members"`, `"status"`. Поле `"members"` — список имён. Построй словарь `active_projects`, в котором ключ — название проекта, а значение — количество участников. В словарь должны попасть только проекты со статусом `"active"` и числом участников от 2 до 5 включительно. Используй одно выражение dict comprehension.

## Пример входных данных
```
projects = [
    {"title": "Docs", "members": ["Анна"], "status": "active"},
    {"title": "Parser", "members": ["Илья", "Рита", "Кирилл"], "status": "active"},
    {"title": "ETL", "members": ["Оля", "Марк", "Сева", "Катя", "Том"], "status": "paused"}
]
```

## Пример выходных данных
```
{'Parser': 3}
```

## Критерии успешного выполнения
- Словарь `active_projects` создаётся одним dict comprehension.
- В него попадают только активные проекты с 2–5 участниками.
- Значением является количество людей в списке `members`.
- Исходный список `projects` не изменяется.

## Подсказки
- Проходи по списку так: `for project in projects` и вычисляй `len(project["members"])`.
- Фильтр по статусу и количеству участников ставь в конце comprehension.
- Следи, чтобы списки в файле не превышали пяти пунктов.

## Частые ошибки и ограничения
- Не оборачивай выражение в `dict()` и не добавляй внешний цикл.
- Не забудь проверить нижнюю и верхнюю границы количества участников.
- Не модифицируй списки `members` внутри исходных словарей.
